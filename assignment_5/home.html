External vs. Internal vs. Inline CSS

Inline styles are applied by the use of the style attribute in an HTML tag, with the value of the attribute being the CSS file. Internal styles are applied to an HTML page through the use of the style tag, with CSS placed between the opening and closing tags. In an HTML page, external styles are applied by using the link tag, which has a href property that links to the URL of a CSS file that is included in the page.
The great differentiator between inline CSS and external stylesheets is a very simple fact. Inline CSS being what it is makes usability impossible, while external sheets can be reused again and again only being constrained by the layout of the HTML document. Styling web pages using inline CSS hinders scalability, readability and maintainability. Inline CSS isn't totally useless though, there is a place and time for fast prototyping simple web pages. One could also see the possiblity of using inline CSS for styling very unique HTML elements.
Internal CSS provides a balance between the two styling methods, and in my opinion nullifies the utility of inline CSS. Lastly, I would like to express my bias towards external CSS as it has proven to be the most useful method for styling for my particular workflow.
